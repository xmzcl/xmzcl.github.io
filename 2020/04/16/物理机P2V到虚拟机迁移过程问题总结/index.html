物理机迁移到虚拟机的时候，总会遇到各种各样的问题。以下是我趟过的坑，都已填平。拿出来分享一下。

问题一:转换到97%时报”There is no matching kernel modules for kernel /tboot.gz”错.

用linux安装光盘进入修复模式。
blkid /dev/sda1
cat /etc/fstab
查看原来的linux的UUID，用下面的命令替换掉。
tune2fs -U c1b9d5a2-f162-11cf-9ece-0020afc76f16 /dev/sda1

问题二:转换linux时，一直停留在1%

converter需设置一个同段辅助IP地址。

问题三:提示无法转换成虚拟机。

P2V转换时，需使用相同版本的VMware-converter，如果已安装不同版本，需卸载低版本的，再安装高版本。

问题四:忘了是什么问题了，不过记得关防火墙就对了。

P2V时关闭防火墙

问题五:容量太大，一次同步不完。可以设置分批同步

P2V设置定时同步数据。

问题六:服务器硬盘没有迁移服务器的大

可以设置最小化硬盘迁移。
