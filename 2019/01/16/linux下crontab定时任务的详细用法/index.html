linux下crontab定时任务的详细用法

在linux下你可以使用crontab周期性的执行任务。crontab设置也非常灵活，下面一起来学习一下。

crontab的安装

crontab基本很多linux上都已经有安装了，直接运行crontab看看是否返回，command not found如果是就执行下面的安装命令。

crontab的启动、停止

配置crontab

crontab在/etc目录下面存在cron.d，cron.daily，cron.weekly,cron.monthly,cron.hourly五个目录和crontab,cron.deny二个文件。
cron.daily是每天执行一次的job，cron.weekly是每个星期执行一次的job.
cron.monthly是每月执行一次的job,cron.hourly是每个小时执行一次的job.
cron.d是系统自动定期需要做的任务，但是又不是按小时，按天，按星期，按月来执行的，那么就放在这个目录下面。
如果是按小时，按天，按星期，按月的来执行的话，则可以放到前面相应的目录下面去。

全局运行
/etc/crontab

单人运行
/var/spool/cron/crontabs/用户名

根据你的需要，编辑以上的crontab文件。

crontab基本格式 :
*　　*　　*　　*　　*　　command

分　时　日　月　周　命令
第1列表示分钟1～59 每分钟用*或者 */1表示
第2列表示小时1～23（0表示0点）
第3列表示日期1～31
第4列表示月份1～12
第5列标识号星期0～6（0表示星期天）
第6列要运行的命令

好像有点晕是不是？下面来点实例就明白了。

例1：每晚的21:30重启apache。

例2：每月1、10、22日的4 : 45重启apache。

例3：每周六、周日的1 : 10重启apache。

例4：每天18 : 00至23 : 00之间每隔30分钟重启apache。

例5：每星期六的11 : 00 pm重启apache。

例6：每一小时重启apache

例7：晚上11点到早上7点之间，每隔一小时重启apache。

例8：每月的4号与每周一到周三的11点重启apache。

例9：一月一号的4点重启apache。

例10：每10分钟执行一次任务。

例11：在某个月的第一个周日执行任务。

例12：每个30秒执行一次任务。

例13：按年、月、周、日、小时执行一次任务。

@yearly 类似于“0 0 1 1 *”。它会在每年的第一分钟内执行，通常我们可以用这个发送新年的问候。

@monthly 代表每月。

@weekly 代表每周。

@daily 代表每日。

@hourly 代表每小时。
